# 📊 Отчет о тестировании проекта SkladProject (Fullstack)

Этот проект полностью покрыт автоматическими тестами на уровне бэкенда (DRF) и фронтенда (Angular).

---

## 🐍 Backend: Django Rest Framework + Pytest
**Стек:** Python 3.12, Pytest, Factory Boy, Mock.

### Проверенные этапы:
1.  **Core Models (SQL):** Проверка создания товаров, складов и связей (ForeignKey). Тестирование каскадного удаления (CASCADE).
2.  **Signals & NoSQL Audit:** Тестирование системы аудита. С помощью **Mock-объектов** проверена отправка логов в MongoDB при создании записей в Django.
3.  **Security & JWT:** Тестирование выдачи токенов через  и проверка прав доступа (IsAuthenticated).
4.  **Business Analytics:** Проверка сложных SQL-запросов для расчета остатков (Приход - Расход).

**Результат:** Все тесты пройдены. HTML-отчеты доступны в папке `tests_reports/`.

---

## 🅰️ Frontend: Angular 21 + Vitest
**Стек:** Angular 19/21, Signals, Vitest, JSDOM.

### Что было проверено (StocksComponent):
*   **Initialization (Lifecycle):** Проверено, что при входе пользователя на страницу Складов компонент автоматически отправляет GET-запрос к API.
*   **Angular Signals (Reactivity):** Проверено, что полученные данные корректно записываются в реактивный сигнал `stocks`.
*   **Data Integrity:** Проверено соответствие имен складов и ID (Mapping) между JSON-ответом бэкенда и UI-компонентом.

**Техническая особенность:** Тестирование реализовано на **Vitest** в среде Node.js (JSDOM), что обеспечивает высокую скорость выполнения по сравнению с Karma. Для стабильности тестов использовано переопределение ресурсов (`overrideComponent`).

---

## 🏆 Итог
Система проверена на целостность данных от базы данных до отображения в браузере. Все критические узлы (авторизация, расчеты, логирование) защищены тестами.

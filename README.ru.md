## 🚀 Обзор проекта
**Sklad PRO AI** — это экосистема управления складом профессионального уровня. Изначально созданный на Vue 3, этот репозиторий представляет собой полную миграцию на **Angular 21**, с упором на высокопроизводительную реактивность, строгую типизацию и модульную архитектуру.

Система управляет запасами, перемещениями между складами и поступлениями товаров, интегрируясь с бэкендом на **Python DRF** и **Gemini AI** для анализа инвентаризации в реальном времени.

---

## 🛠 Технологический стек
-   **Frontend Framework**: Angular 21 (Standalone-архитектура)
-   **Управление состоянием**: Angular Signals (точечная реактивность)
-   **Безопасность**: JWT Auth с механизмом обязательной смены пароля
-   **Стилизация**: Bootstrap 5.3 + SCSS (оригинальная SVG-иконографика)
-   **Связь с API**: HttpClient с использованием RxJS и функциональных интерцепторов
-   **ИИ-интеграция**: Кастомный модуль ИИ для рекомендаций по остаткам

---

## 🏗 Архитектурные особенности (Senior-подход)
Проект был мигрирован с мышлением "Enterprise-First":

1.  **Signals вместо Observables**: Использование **Angular Signals** для состояния UI, что сокращает циклы проверки изменений (change detection) и повышает производительность.
2.  **Обязательный флоу безопасности**: Реализация системы активации через "Временный пароль". Пользователи вынуждены менять пароль при первом входе через защищенный переход в интерфейсе под управлением интерцептора.
3.  **Продвинутое управление персоналом**: Выделенный модуль администратора, где регистрация сотрудников происходит через раздельные поля **Логин** и **Полное имя**, с автоматической генерацией временного пароля.
4.  **Функциональные интерцепторы**: Глобальный `AuthInterceptor` обрабатывает внедрение JWT-токенов и обработку ошибок 401/403.
5.  **Clean Code & SOLID**: Строгое разделение между Core-сервисами, компонентами фич (Features) и общими элементами интерфейса (Shared).

## 🤖 Функции ИИ
В систему встроен выделенный **ИИ-аналитик**, который:
*   Анализирует текущие уровни запасов на всех складах.
*   Предоставляет рекомендации по пополнению на основе истории перемещений.
*   Корректно обрабатывает сбои ИИ-сервисов, не прерывая работу основного складского функционала.

## 📦 Установка и настройка

1.  **Клонируйте репозиторий:**
    ```bash
    git clone https://github.com
    cd sklad-pro-angular
    ```

2.  **Установите зависимости:**
    ```bash
    npm install
    ```

3.  **Запустите сервер для разработки:**
    ```bash
    npm start
    ```
    Приложение будет доступно по адресу `http://localhost:4200/`.

## 🔄 Процесс миграции: из Vue в Angular
Этот проект демонстрирует способность переносить сложную логику между фреймворками:
*   **Реактивность**: Миграция с Vue `ref/computed` на Angular `signal/computed`.
*   **Директивы**: Переход с `v-if/v-for` на современный синтаксис управления потоком Angular `@if/@for`.
*   **Слой сервисов**: Замена Axios на Angular `HttpClient` с использованием Dependency Injection для лучшей тестируемости.
